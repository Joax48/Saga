# Migration: npm → pnpm

**Date:** 2026-05-28  
**Branch:** `feature/profile-detail-connection`

---

## For teammates: switching to this branch

Run these steps once after pulling this branch. After that, el flujo normal de trabajo no cambia.

### 1. Install pnpm globally (if you don't have it)

```bash
npm install -g pnpm
```

Verify: `pnpm --version` should print `11.x.x`.

### 2. Delete the old `node_modules`

The `node_modules` generated by npm is incompatible with pnpm. Delete it before installing:

```bash
# PowerShell (Windows)
Remove-Item -Recurse -Force node_modules

# or with make
make clean
```

### 3. Install dependencies with pnpm

```bash
pnpm install
```

### 4. Done — use the same commands as before

```bash
pnpm run dev:api   # API on port 3001
pnpm run dev:web   # Web on port 3000
```

> **Important:** from this point forward, always use `pnpm` instead of `npm`. Running `npm install` on this project will generate a `package-lock.json` and produce a different `node_modules` — both are gitignored and will cause inconsistencies with the rest of the team.

---

## Why

npm was replaced by pnpm for two reasons:

1. **Security:** npm's permissive dependency hoisting allows packages to access dependencies they never declared, enabling supply-chain attacks through phantom dependencies.
2. **Performance and efficiency:** pnpm uses a content-addressable store — packages are stored once globally and linked, reducing disk usage and install times by 2–3x in a monorepo.

## OneDrive note

The project lives inside an OneDrive-synced folder. pnpm's default layout uses symlinks in `node_modules/.pnpm/` which OneDrive does not sync correctly (this caused `.next` errors previously). The `.npmrc` setting `node-linker=hoisted` makes pnpm use a flat `node_modules` layout (identical to npm's behavior), eliminating symlinks while keeping all other pnpm benefits.

## Working with pnpm going forward

### Install all dependencies

```bash
pnpm install
```

### Add a dependency to a specific app

```bash
# Production dependency
pnpm --filter api add oracledb
pnpm --filter web add react-query

# Dev dependency
pnpm --filter api add -D @types/node
```

### Add a dependency to the root (shared tooling)

```bash
pnpm add -D prettier -w
```

### Remove a dependency

```bash
pnpm --filter api remove some-package
```

### Run a script in a specific app

```bash
pnpm --filter api test
pnpm --filter web build
```

### Run scripts in all apps at once

```bash
pnpm -r run lint
pnpm -r run typecheck
```

## CI/CD impact

The GitLab CI pipeline (`.gitlab-ci.yml`) was updated to:

1. Install pnpm globally via `npm install -g pnpm@11` in `before_script`.
2. Cache the pnpm store at `$PNPM_HOME` (`.pnpm-store/`) instead of `.npm/`.
3. Use `pnpm install --frozen-lockfile` in the install stage (equivalent to `npm ci`).
4. Replace all `npm run` with `pnpm run` in job scripts.
